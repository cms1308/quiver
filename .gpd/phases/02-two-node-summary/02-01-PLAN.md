  <name>Task 1: Audit DB and extract complete class-level data</name>
  <files>results/two_node_tables.py</files>
  <action>
    Write a self-contained Python script that reads quivers.db and produces the complete two-node classification.

    Step 1 -- DB Audit:
    Query both universality_class and theory tables to build a complete picture of each class:
    - class_id, gauge_pair, rank0_mult, rank1_mult
    - a_over_N2 (from universality_class -- all 326 have this)
    - c_over_N2: use class-level c_exact if available (181 classes); otherwise use theory-level c_over_N2 from the representative theory (rep_theory_id). ALL 326 classes have c available through one of these paths.
    - a_over_c: from universality_class. Three tiers:
      * 285 classes with a > 0: a/c defined and in [1/2, 3/2] (unitary SCFTs)
      * 10 classes with a = 0: set a_over_c = None, flag as Type-I (a=c=0)
      * 31 classes with a < 0: a/c defined but may violate HM bounds, flag as non-unitary
    - R-charges: use R_exact if available; otherwise use R_numerical from rep theory.
    - n_theories count per class
    - Representative matter content (matter0, matter1, edges from rep theory)
    - veneziano_any, veneziano_all flags

    Step 2 -- Numerical c extraction for classes missing c_exact:
    For each of the 145 classes without c_exact:
    - Query the representative theory's c_over_N2 (all exist per DB audit)
    - If c_over_N2 is available from the rep theory, use it
    - Cross-check: verify c = a / (a/c) matches c_over_N2 from theory table (for classes with a_over_c != NULL). This works for both positive and negative a — the relation c = a/(a/c) holds regardless of sign.
    - Flag whether data is exact_symbolic or numerical_only

    Step 3 -- Type-I handling:
    For the 10 classes with |a_over_N2| < 1e-8:
    - Verify |c_over_N2| < 1e-8 (both genuinely zero)
    - Set a_over_c = None with label "N/A (a=c=0, Type-I)"
    - Set tier = "type_I", unitary = null (indeterminate)

    Step 3b -- Non-unitary classification:
    For the 31 classes with a_over_N2 < 0:
    - Flag as tier = "non_unitary", unitary = false
    - Record whether c_over_N2 < 0 (25 classes) or c_over_N2 > 0 (6 classes)
    - Record whether a/c violates HM bounds 1/2 <= a/c <= 3/2 (13 classes violate)
    - These theories are asymptotically free but do NOT flow to unitary SCFTs (a-theorem requires a > 0)
    - Still record full superconformal data — the a-maximization result is mathematically valid even if the IR theory is non-unitary

    Step 3c -- Unitary classification:
    For the remaining 285 classes with a_over_N2 > 0:
    - Set tier = "unitary", unitary = true
    - All should satisfy HM bounds 1/2 <= a/c <= 3/2

    Step 4 -- Summary statistics:
    Compute per gauge pair type AND per rank sector:
    - Number of classes
    - Number with exact symbolic data
    - Range of a/N^2, c/N^2, a/c
    - Mean a/c (excluding Type-I and non-unitary)
    - Number of Type-I (a=0) classes
    - Number of non-unitary (a < 0) classes and non-unitary fraction
    - Number of unitary (a > 0) classes

    Convention check: use CLAUDE.md conventions throughout. Gauge pair ordering alphabetical. Rank multiplier notation rank(m0,m1).
  </action>

All numerical consistency checks pass.</done>
